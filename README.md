## DesignPattern 设计模式
### 设计模式的原则
* 单一职责原则
* 开闭原则：对扩展开放 对修改关闭
* 里氏替换原则
* 依赖倒置原则 声明时多用接口、抽象类而不是具体子类去声明
* 接口隔离原则 每个接口承担自己的责任
* 迪米特/最少知识原则
